HTML

  • "head" contains information the user cannot see (except for "title")
  • "meta" can specify character set, page description, keywords, author, and viewport settings. Attributes include "charset," "content," "http-equiv," and "name." Examples: meta name="keywords" content="HTML, CSS, JavaScript"; meta name="description" content="Free Web Tutorials for HTML"; meta name="author" content="Jordan Barringer". To refresh the document every 30 seconds, 'meta http-equiv="refresh" content="30"'. To set the viewport to make the site look good on all devices, 'meta name="viewport" content="width-=device-width, initial-scale=1.0"'.
  • "title" is required for Search Engine Optimization (SEO) and is the tab name at the top of the browser.
  • "style" defines CSS styles for the site
  • "base" defines the base URL.
  • "link" connects an external source.
  • "script" embeds client-side scripts.
  • "body" contains information the user can see.
  • "h1", "h2", "h3", "h4", "h5", and "h6" are headers from largest to smallest. h1 should only appear once per page.
  • "p" represents a paragraph.
  • "ul" is an unordered list.
  • "ol" is an ordered list.
  • "li" are list items
  • "img" contains info about images. Elements include "src" (the location of the image) and "alt" (name of the image; imperative for people using screen readers).
  • "br" creates a line of empty space
  • "a"nchor element; creates links to the same webpage or other pages. The "href" attribute points to the URL for the link.
  • "header" appears at the top of the page, and usually contains the nav menu and/or introductory content.
  • "main" is the main body of the text.
  • "footer" appears at the bottom of the page and usually contains author, copyright, contact, sitemap, and navigation.
  • "section" defines a section in a document that contains a thematic element.

CSS

  • Cascading Style Sheets (CSS) define colors, aesthetics, visual layout, and organization.
  • Inline CSS - apply to a specific HTML tag. Useful for quick and permanent changes, but less flexible than internal and external style sheets.
  • Internal Style Sheet - an imbedded style sheet in the head section of the HTML file. The rules apply to the whole page, and you can configure classes & IDs.
  • External Style Sheet - a separate document of code added to the HTML document within head with a link to the stylesheet.
  • margin - indicates how much spave we want around the outside of an element
  • padding - indicates how much space we want around the inside of an element.
  • "*" declares all elements on the page.
  • You can define 2 selectors using a comma and the ENTER key.
    EX: header,
    footer {
    ...
    }
  • Do Not Repeat Yourself (DRY) - reduces the number of lines of code that need to be written/maintained.
  • display: block - assigns the selector to start on a new line
  • Classes are identified by a '.' before the class name, whereas ID's are identified by '#'.

Git

  • "git add -A" adds your changes to a buffer feature.
  • "git commit -m"(insert message regarding changes here)"" saves the changes to the buffer. Add a message to each change to keep track of what has been changed.
  • "git push" pushes your changes to the github branch.
  • "git status" checks what branch we are currently in.
  • "git checkout -b (branch-name)" creates a new branch and switches to it.
  • "git clone (SSH link of your github project)" clones the project into your personal files.
  • "git pull origin main" ensures that the project in your personal files is up to date with the project in github.
  • "git checkout main" places you in the main branch

JavaScript

  • var - variable
  • var shape(name) = "circle"(data/string)
  • script src = "(input script file location)" - this goes in the footer of your html document to link .js to .html
  • To open Chrome Devtools, rightclick anywhere in the browser and hit "inspect" OR ctl+shift+I
  • Control Flow is ordering commands top to bottom to ensure the code runs smoothly
  • if statement - a conditional statement that will only execute if the statement meets a condition, or is true. EX: if (condition) {
    // block of code to be executed if condition is truthy
    }
  • A value is considered falsy if it is 0, -0, On, "", null, undefined, or NaN, or a Boolean data type with the "false" value.
  • if (first condition) {
    // block of code to be executed if first condition is truthy
    } else if (second condition) {
    // block of code to be executed if second condition is truthy
    }
  • console.log() - tells the console to display whatever is in the parenthesis.
  • array - a single variable that holds a group of data. Ex: var shapes = ["triangle", "square", "pentagon", "circle"];
    OR
    studentInfo = ["Lu", 54, true]
  • The items listed within an array are numbered 0, 1, 2, 3... meaning the value of "triangle" or "Lu" would be 0, while "pentagon" and "true" would be 2.
  • for loops allow a single block of code to be tested over and over.
    Ex: for (var x = 0; x < shapes.length; x++) {
    console.log(shapes([x]);
    }
  • To comment out code in JavaScript, surround the code with /* and */
  • function - set of instructions that tells the computer what to do to define and reuse code. You must first define a function
    Ex: function makeTea () {
    // code block
    }
    and then call the function
