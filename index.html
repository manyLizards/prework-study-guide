Prework Study Guide

HTML

  • "head" contains information the user cannot see (except for "title")
  • "meta" can specify character set, page description, keywords, author, and viewport settings. Attributes include "charset," "content," "http-equiv," and "name." Examples: meta name="keywords" content="HTML, CSS, JavaScript"; meta name="description" content="Free Web Tutorials for HTML"; meta name="author" content="Jordan Barringer". To refresh the document every 30 seconds, 'meta http-equiv="refresh" content="30"'. To set the viewport to make the site look good on all devices, 'meta name="viewport" content="width-=device-width, initial-scale=1.0"'.
  • "title" is required for Search Engine Optimization (SEO) and is the tab name at the top of the browser.
  • "style" defines CSS styles for the site
  • "base" defines the base URL.
  • "link" connects an external source.
  • "script" embeds client-side scripts.
  • "body" contains information the user can see.
  • "h1", "h2", "h3", "h4", "h5", and "h6" are headers from largest to smallest. h1 should only appear once per page.
  • "p" represents a paragraph.
  • "ul" is an unordered list.
  • "ol" is an ordered list.
  • "li" are list items
  • "img" contains info about images. Elements include "src" (the location of the image) and "alt" (name of the image; imperative for people using screen readers).
  • "br" creates a line of empty space
  • "a"nchor element; creates links to the same webpage or other pages. The "href" attribute points to the URL for the link.
  • "header" appears at the top of the page, and usually contains the nav menu and/or introductory content.
  • "main" is the main body of the text.
  • "footer" appears at the bottom of the page and usually contains author, copyright, contact, sitemap, and navigation.
  • "section" defines a section in a document that contains a thematic element.
  • "git add -A" adds your changes to a buffer feature.
  • "git commit -m"(insert message regarding changes here)"" saves the changes to the buffer. Add a message to each change to keep track of what has been changed.
  • "git push" pushes your changes to the github branch.
  • "git status" checks what branch we are currently in.
  • "git checkout -b (branch-name)" creates a new branch and switches to it.
  • "git clone (SSH link of your github project)" clones the project into your personal files.
  • "git pull origin main" ensures that the project in your personal files is up to date with the project in github.
  • "git checkout main" places you in the main branch
